### Services Built

| Service | Purpose | Key Features |
|---------|---------|--------------|
| `embedding_service.py` | Extract image features | 1280-dim EfficientNet-B0, L2 normalization, GPU support |
| `tfidf_service.py` | Text vectorization | Single fitted vectorizer, consistent vocabulary, sparse representation |
| `image_service.py` | File management | Auto-sync with MongoDB, rollback on failure, organized storage |
| `recommendation_service.py` | Scoring | Hybrid algorithm, top-K selection, cosine similarity |

### Critical Design Features

1. **Filename Synchronization**
   - Single function: `generate_image_record()` in `filename_utils.py`
   - Generated once, used as both DB key AND filesystem filename
   - Ensures perfect synchronization

2. **TF-IDF Consistency**
   - Vectorizer fitted ONCE on representative corpus
   - Saved to `ml/vectorizer.pkl`
   - Loaded at startup, never refitted
   - All products use same vocabulary

3. **Background Processing**
   - Product upload returns immediately (fast user experience)
   - Embeddings generated in background
   - Status: "processing" → "available"
   - Buyers only see completed products

4. **Hybrid Scoring**
   - Image similarity: cosine distance in 1280-dim EfficientNet space
   - Text similarity: cosine distance in TF-IDF space
   - Hybrid = 70% image + 30% text
   - Configurable weights per request

## Example Workflow

### Seller adds a product:
```bash
1. Register seller
   POST /api/seller/register
   → seller_id = "abc123"

2. Upload product with 3 images
   POST /api/seller/product
   → product_id = "xyz789"
   → status = "processing"
   
3. Background task:
   - Extracts 1280-dim embedding for each image
   - Generates TF-IDF from description
   - Updates product status to "available"
   - Takes ~30-60 seconds

4. Product now searchable
```

### Buyer searches for similar dresses:
```bash
1. Upload query image
   POST /api/buyer/recommend
   
2. Backend processes:
   - Extracts query embedding (50-100ms)
   - Generates query TF-IDF (5ms)
   - Queries all available products
   - Computes hybrid scores for each
   - Ranks by hybrid_score
   - Returns top-K results (200-400ms total)

3. Results include:
   - Product details
   - Image similarity score
   - Text similarity score
   - Hybrid score
   - Product images with URLs
```
